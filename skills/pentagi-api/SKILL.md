# PentAGI API Reference

The PentAGI backend exposes both REST and GraphQL APIs. All endpoint are under
`/api/v1/` and require Bearer token authentication.

## Authentication

```bash
# Set these once
PENTAGI_URL="https://<server>:8443"
TOKEN="<api-token>"
AUTH="Authorization: Bearer $TOKEN"
```

Generate tokens in the PentAGI UI: Settings → API Tokens.

## REST API Endpoints

### Flows (pentest engagements)

```bash
# List all flows
curl -H "$AUTH" $PENTAGI_URL/api/v1/flows

# Create a new flow
curl -X POST $PENTAGI_URL/api/v1/flows \
  -H "$AUTH" \
  -H "Content-Type: application/json" \
  -d '{"input": "scan example.com for open ports and web vulnerabilities", "provider": "openai"}'

# Get a specific flow
curl $PENTAGI_URL/api/v1/flows/<flow-id> -H "$AUTH"

# Get flow results
curl $PENTAGI_URL/api/v1/flows/<flow-id>/results -H "$AUTH"

# Delete a flow
curl -X DELETE $PENTAGI_URL/api/v1/flows/<flow-id> -H "$AUTH"
```

### Flow Files (artifacts from tool runs)

```bash
# List files generated by a flow
curl $PENTAGI_URL/api/v1/flows/<flow-id>/files -H "$AUTH"

# Download a specific file
curl $PENTAGI_URL/api/v1/flows/<flow-id>/files/<file-id> -H "$AUTH"
```

### API Tokens (manage programmatic access)

```bash
# List tokens
curl $PENTAGI_URL/api/v1/tokens -H "$AUTH"

# Create token
curl -X POST $PENTAGI_URL/api/v1/tokens \
  -H "$AUTH" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-agent", "scope": "flows:read flows:write"}'
```

## GraphQL API

Endpoint: `POST $PENTAGI_URL/api/v1/graphql`

```bash
# Query flows
curl -X POST $PENTAGI_URL/api/v1/graphql \
  -H "$AUTH" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "{ flows { id title status createdAt } }"
  }'

# Create flow via mutation
curl -X POST $PENTAGI_URL/api/v1/graphql \
  -H "$AUTH" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "mutation($provider: String!, $input: String!) {
      createFlow(modelProvider: $provider, input: $input) { id title status }
    }",
    "variables": {
      "provider": "openai",
      "input": "recon and vulnerability scan example.com"
    }
  }'
```

## LLM Providers

Pass the provider name when creating flows:

| Provider key | Notes |
|---|---|
| `openai` | Requires OPEN_AI_KEY in .env |
| `anthropic` | Requires ANTHROPIC_API_KEY in .env |
| `gemini` | Requires GEMINI_API_KEY in .env |
| `deepseek` | Requires DEEPSEEK_API_KEY in .env |
| `openrouter` | Requires OpenRouter config in examples/configs/ |
| `ollama` | Points to OLLAMA_SERVER_URL |
| `custom` | Uses LLM_SERVER_URL |

## Flow Status Lifecycle

```
pending → running → completed
                      → failed
                      → cancelled
```

Poll `GET /api/v1/flows/<flow-id>` until `status` is `completed`, `failed`, or
`cancelled`. On completion, retrieve results from `/flows/<flow-id>/results`.

## Python Client

```python
import requests, time

client = requests.Session()
client.headers.update({"Authorization": f"Bearer {TOKEN}"})
BASE = f"{PENTAGI_URL}/api/v1"

def create_flow(target: str, provider: str = "openai") -> dict:
    r = client.post(f"{BASE}/flows", json={"input": target, "provider": provider})
    return r.json()

def wait_for_flow(flow_id: str, poll_interval: int = 5) -> dict:
    while True:
        flow = client.get(f"{BASE}/flows/{flow_id}").json()
        if flow["status"] in ("completed", "failed", "cancelled"):
            return flow
        time.sleep(poll_interval)

def get_results(flow_id: str) -> dict:
    return client.get(f"{BASE}/flows/{flow_id}/results").json()
```
